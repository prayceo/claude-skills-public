# Investor Deck Looks — Taxonomy Reference

A condensed reference drawn from the "Investor Deck Looks: Comprehensive Master Catalog" — a working taxonomy of the chart, dashboard, and slide formats used to run consumer subscription, advertising, and SaaS businesses. Use this to recommend *how* a Council decision or metric should be visualized when it will be presented to the board, investors, or another external audience — not just *what* the decision is.

Three layers, from most to least polished:
- **Board / Investor** — one dominant chart, sparse copy, one strategic callout. No dense BI screenshots.
- **Executive Operating** — multi-panel dashboards with owner, cadence, metric definitions, decision rules.
- **Team Workflow** — requirements lists, rough screenshots, raw evidence. Never board-ready.

## Board-Ready Looks (recommend these for external audiences)

| Look | Best For |
|---|---|
| Executive Growth Narrative | Revenue, subscriber/customer growth, ARR, EBITDA, acquisition efficiency — one headline metric |
| Financial Operating Table | Budget, forecast, P&L, margin, cash, ARR bridge (beginning/new/expansion/churn/ending) |
| Funnel And Conversion View | Acquisition-to-paid stages, activation, paywall conversion |
| Renewal And Retention Cohort View | Renewal rates, NRR/GRR, cohort survival, revenue quality by vintage |
| Unit Economics Explainer View | CAC payback, magic number, LTV/CAC — chart plus the formula spelled out under the title |
| Budget Vs. Actual Pacing View | Actual vs. budget/target by period, with a variance callout |
| Headcount & Org Growth View | Headcount by department, hiring pace vs. plan |
| Investor Benchmark / Market Data Look | Peer comps, valuation multiples, market benchmarks, customer concentration |
| Segment Analysis View | Breaking one average into groups — user segments or product-line segments |
| Multi-Panel KPI Dashboard | A full operating snapshot across a related metric family, for an exec owner |
| Ad-Supported / Creator Monetization View | Ad revenue, account/revenue retention, ACV |
| Jobs-To-Be-Done / User State Map | Product strategy narrative — why the customer comes, what moment is served |

## Internal-Only Looks (never present these to the board as-is)

Analytics Requirements List, Rough Dashboard Screenshot With Notes, Raw Chart / Workbook Evidence, Screenshot-In-Frame Demo, Section Divider / Placeholder, Reference Article / Thought Leadership Capture, Content Supply And Content Demand View, Engagement Depth View, Power User Curve. These belong in the team-workflow layer — good for the analyst backlog, wrong for a board memo.

## How To Use This

When a Council decision will be presented externally (board, investors, press), name the matching Board-Ready look and say why: *"This should become an Executive Growth Narrative — one chart, one callout, no dense tables."* If the underlying data most naturally fits an Internal-Only look, flag that it needs to be translated up a layer before it reaches the board — don't let a rough Amplitude screenshot go straight into a board deck.
