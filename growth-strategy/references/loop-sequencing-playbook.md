# Loop Sequencing Playbook


---

## The Three Questions of Loop Sequencing

Every company at every stage should be able to answer:

1. **What is our primary growth loop right now?** (name every step, every conversion rate)
2. **Where is its invisible asymptote?** (what will make it plateau?)
3. **What is the next loop, and what minimum viable scale of it are we building today?**

If you can't answer all three, you are reactive to growth, not strategic about it.

---

## Step 1: Diagnose Your Current Loop

**Map every step explicitly:**
```
[Input] → [Step 1] → [Step 2] → [Step 3] → [Output] → [Reinvestment back to Input]
```

For each step, identify:
- What is the conversion rate? (% of users who complete this step)
- What is the cycle time? (how long does one full loop cycle take?)
- What is the reinvestment rate? (% of output that feeds back into input)

**Example: Content SEO loop**
```
[User generates content] 
  → [Content indexed by search engine] 
  → [New user discovers via search] (conversion rate: 0.5–2% of searchers)
  → [New user creates account] (conversion rate: 20–40% of page visitors)
  → [New user generates more content] (conversion rate: 10–30% of new users)
  → [More content indexed] (reinvestment rate: depends on content quality/volume)
```

**Identify the bottleneck:** The step with the lowest conversion rate is the highest leverage improvement opportunity. A 2× improvement there compounds through the whole loop.

## Step 2: Identify the Asymptote

**Warning signs a loop is approaching its ceiling:**

| Signal | What it means |
|--------|--------------|
| Top experiments return < 5% improvement | Optimization ceiling reached |
| CAC rising despite same channel mix | Audience saturation |
| Viral coefficient declining over time | Social capital loop fading |
| Cohort retention curves declining | Core product value eroding |
| SEO traffic plateauing | Content saturation in category |
| Referral rate declining | Novelty wearing off; market penetration high |

**Structural asymptotes by loop type:**

- **Social capital:** Saturates the community of status-seeking early adopters in your category
- **Referral:** Saturates the social graph of your current users
- **Content/SEO:** Saturates keyword space in your category; search algorithm changes
- **Direct NE:** Saturates the community (company, city, professional group) where NE operates
- **Cross-side NE:** Saturates market on one or both sides
- **Paid:** Budget ceiling, targeting saturation, creative fatigue

## Step 3: Choose the Next Loop

**Selection criteria:**
1. **What scale do you have now that you didn't have at founding?** (UGC volume, users, revenue, brand)
2. **What structural shifts in your market have opened new loops?** (platform changes, API access, category maturity)
3. **What does your current loop's output produce that could seed the next loop?**
4. **How long will the next loop take to ignite?** (match timing to when current loop plateaus)

**Common sequencing patterns:**

### Social Capital → Content/SEO
Works when: Social capital loop has generated enough users to produce meaningful UGC volume.
Transition: Build distribution infrastructure for user content (SEO optimization, content pages, sharing surfaces).
Timing: Start 12–18 months before social capital loop plateaus.

### Social Capital → Referral + Product-Embedded Sharing
Works when: Core product has a natural shareable artifact.
Transition: Build sharing into the core workflow (not a separate "refer a friend" button).
Timing: Build this into the product before social capital loop fades.

### Content/SEO → Direct Network Effects
Works when: Content loop has brought in enough users to create critical mass for a community/network feature.
Transition: Launch community features (groups, follower graphs, messaging) once user base is large enough to feel alive.
Timing: Introduce community features when DAU/MAU is high enough that users will find community active.

### Direct NE → Cross-Side NE
Works when: Primary user type has enough density that a secondary user type can be pulled in.
Transition: Build features that create artifacts the secondary user type needs (Figma: share link for non-designers).
Timing: Once within-segment density is high enough that secondary users reliably encounter primary users.

### Any Loop → Platform/Ecosystem
Works when: User base is large enough that 3rd party builders can earn revenue building on top.
Transition: Open APIs, plugin architecture, developer program, marketplace.
Timing: Need critical mass before ecosystem can sustain itself; too early = ghost town; requires significant investment.

## Step 4: Build the Bridge

The hardest part of loop sequencing is the transition period — when the current loop is plateauing and the next loop hasn't ignited yet. This is the trough of growth.

**Principles for navigating the bridge:**

1. **Overlap, don't replace.** The new loop should be planted while the current loop is still growing, not after it's plateauing. Most companies start the next loop too late.

2. **The new loop requires different capabilities.** Content loop needs editorial/SEO infrastructure. Network effects loop needs community management. Platform loop needs developer relations. Start building the capability before you need the loop to perform.

3. **Metrics change.** The north star for a social capital loop (referrals, K-factor) is different from a content loop (organic search traffic, UGC volume) which is different from a network effects loop (density, connections per user). Plan the metric transition before the team is confused.

4. **Don't kill the current loop prematurely.** Even a capped loop with low growth still produces users. Keep it running while seeding the next one. Only deprioritize when the new loop is generating more efficient users.

## Loop Sequencing for your product — Detailed Hypothesis

**Current state:**
- Primary loop: App store + paid social → install → activation → daily habit → some social sharing
- Likely asymptote signals: If paid CAC is rising, organic is flat, viral coefficient is low → the current loop is approaching its ceiling

**Proposed sequence:**

### Phase 1 — Deepen Retention Loop (Now)
Goal: Raise LTV by improving daily habit formation and streak completion.
- People loop: Personalized onboarding with a human touch for first week
- Product loop: Streak mechanics, personalized content selection, progress milestones
- Incentives loop: Social sharing of personal milestones (not points — appropriate)
- Messaging loop: Smart push notification timing based on user's daily session time pattern

This raises LTV, which unlocks larger CAC budgets for Phase 2.

### Phase 2 — Build Content/SEO Loop (6–12 months)
Goal: Create engagement session content pages that rank for high-intent queries.
- Identify top 1,000+ content intent queries ("content for anxiety," "morning engagement session for couples," "content for job interview")
- Create or curate best-in-class content for each
- Each page converts intent → sign-up → subscriber
- This creates a data content loop: more content → more SEO → more users → user engagement/reviews improve content

This creates near-zero marginal CAC at scale.

### Phase 3 — Community Loop (12–24 months)
Goal: Create cross-side network effects between individual users and community communities.
- Individual users → share group engagement sessions with community
- Community leaders → distribute your product to their audience → bulk sign-ups
- Community leaders → get tools to manage group engagement sessions → pull in more individuals
- Cross-side NE: the more communityes, the more individuals; the more individuals, the more community leaders see evidence of value

This creates a B2B2C model with much higher LTV per community relationship.

### Phase 4 — AI Personalization Flywheel (18–36 months)
Goal: Build a data flywheel that makes your product's content increasingly personalized and irreplaceable.
- User usage patterns, topics, time-of-day, streak behavior → training signal
- AI-generated personalized engagement session content that improves with each engagement
- The more users engage, the better the personalization → better retention → more data
- This moat is nearly impossible for new entrants to replicate without the data

## Loop Health Scorecard

For each active loop, track monthly:

| Metric | Description | Red flag |
|--------|-------------|----------|
| Input volume | # of users entering the loop | Declining MoM |
| Step conversion rates | % completing each step | Any step < baseline |
| Cycle time | Days for one full loop cycle | Increasing trend |
| Output volume | # of new inputs generated | Declining ratio to input |
| Reinvestment rate | % of output feeding back in | < 10% |
| Loop velocity | Output ÷ cycle time | Declining trend |

**Loop power index (rough):**
```
LP = (Reinvestment rate × Product of step conversions) ÷ Cycle time (weeks)
```
Higher = more powerful loop. Track directionally over time.

## The Sequencing Timing Framework

```
Current loop health:   Strong (↑)    Plateauing (→)    Declining (↓)

When to plant          Now           Now               Yesterday
next loop:             (early)       (urgent)          (you're late)

Investment level       10–20%        30–50%            50–80%
in next loop:          of resources  of resources      of resources

Risk level:            Low           Medium            High
```

The biggest mistake: waiting until the current loop is in decline before investing in the next loop. By then, you've lost the option value of compounding early.
